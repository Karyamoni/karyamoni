# Karyamoni Product Plan

## 1. Product Summary

Karyamoni is a virtual clothing try-on application for e-commerce merchants. The first priority is to launch as an IKAS application for stores in Turkey, followed later by Shopify and other popular CMS builders.

The main product experience is an embedded 3D virtual fitting cabin inside the merchant's store, where shoppers can preview how clothing may fit before purchasing without leaving the product page flow.

## 2. Problem

E-commerce clothing stores face high return rates because size labels are inconsistent. A medium size from one brand may not fit like a medium size from another brand. Shoppers often buy the wrong size, and merchants lose money through returns, shipping, support work, and customer dissatisfaction.

## 3. Product Goal

Reduce clothing return rates by giving shoppers a more confident try-on experience before checkout.

## 4. Target Users

- Online shoppers buying clothing.
- IKAS store owners selling apparel.
- E-commerce managers responsible for conversion and return rates.
- Customer support teams handling size-related return requests.

## 5. Primary Market

Turkey is the first launch market.

## 6. Language Support

Karyamoni should support:

- Turkish as the primary launch language.
- English as a secondary language for international readiness.

## 7. Platform Strategy

### Phase 1: IKAS Application First

IKAS is the main focus for the first version. Karyamoni should be delivered as an IKAS application that merchants can install and activate with minimal effort. The product should feel like a native extension of the merchant's store, not a separate shopper destination.

### Phase 2: Shopify

After the IKAS version is validated, Karyamoni can expand to Shopify with a similar merchant setup flow and storefront try-on experience.

### Phase 3: Additional CMS Builders

Future platform support can include other popular e-commerce CMS builders after the core product and business model are proven.

## 8. MVP Scope

The first MVP should focus on proving the core value: merchants can install Karyamoni as an IKAS app, activate it on product pages, and shoppers can try on clothing virtually without leaving the store experience.

Suggested MVP features:

- IKAS app installation and activation flow.
- Product-level try-on button or embedded entry point.
- Embedded 3D virtual fitting cabin interface.
- Basic user body/profile input.
- Clothing preview on a virtual avatar or fitting model.
- Turkish and English UI text.
- Merchant dashboard for setup and status.
- Return-rate and usage tracking foundations.

## 9. Core Shopper Flow

1. Shopper opens a product page on an IKAS store.
2. Shopper clicks the Karyamoni try-on entry point.
3. Shopper enters or selects fit-related body information.
4. Karyamoni opens the 3D fitting cabin inside the store experience, such as a modal, drawer, or embedded panel.
5. Shopper previews the clothing item.
6. Shopper closes the try-on experience and continues shopping or purchasing on the same store page.

## 10. Core Merchant Flow

1. Merchant installs the Karyamoni IKAS application.
2. Merchant grants required app permissions and completes initial setup.
3. Merchant configures branding, language, and product settings.
4. Merchant maps clothing products to try-on data.
5. Merchant activates Karyamoni on selected product pages.
6. Merchant reviews usage and return-rate impact.

## 11. Design Direction

Design references will be added later. For now, the product should feel:

- Trustworthy.
- Modern.
- Easy for shoppers.
- Practical for merchants.
- Premium enough for fashion brands.
- Fast enough for everyday e-commerce use.

## 12. Success Metrics

Early product success can be measured by:

- Try-on button click rate.
- Try-on completion rate.
- Add-to-cart rate after try-on.
- Purchase conversion after try-on.
- Size-related return rate reduction.
- Merchant activation rate.
- Merchant retention.

## 13. Key Risks

- 3D try-on accuracy may not be trusted by shoppers.
- IKAS app and storefront extension limitations may shape the MVP.
- Product setup may be too complex for merchants.
- Performance issues may hurt storefront conversion.
- Body data privacy expectations must be handled carefully.

## 14. Open Questions

- What exact IKAS app installation, storefront extension, and product data options are available?
- Will the MVP use a customizable avatar, fixed models, or uploaded user measurements?
- What clothing categories should be supported first?
- What data is required from merchants for each clothing item?
- How accurate does the first try-on experience need to be to create trust?
- What privacy and consent flows are required for body/profile data?
- What design references should guide the 3D cabin and merchant dashboard?

## 15. Immediate Next Steps

1. Collect design references.
2. Define the MVP shopper flow in detail.
3. Define the MVP merchant setup flow.
4. Research IKAS app installation and storefront extension capabilities.
5. Choose the initial clothing category.
6. Draft the technical architecture.
7. Create wireframes for the embedded try-on entry and 3D cabin.
