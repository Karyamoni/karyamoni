# Karyamoni Roadmap

## 1. Roadmap Principle

Karyamoni should move in small validation steps. The first goal is not to build the final version of virtual try-on. The first goal is to prove that shoppers use it, merchants can activate it, and the experience can reduce size-related returns.

## 2. Phase 0: Product Definition

Status: current phase.

Goals:

- Define MVP scope.
- Select first clothing category.
- Collect design references.
- Research IKAS app installation and storefront extension options.
- Define core shopper and merchant flows.
- Draft technical architecture.
- Define privacy and localization requirements.

Deliverables:

- Product plan.
- MVP spec.
- User flows.
- Architecture draft.
- Design brief and wireframes.

## 3. Phase 1: Prototype

Goals:

- Build a clickable or functional prototype of the shopper try-on flow.
- Validate the 3D fitting cabin direction.
- Test Turkish and English copy.
- Demo embedded product-page entry behavior.

Deliverables:

- Product-page try-on entry prototype.
- Body/profile input prototype.
- 3D fitting cabin prototype.
- Fit result screen.
- Merchant setup mock.

Success signals:

- Users understand what Karyamoni does.
- Users understand what information they need to provide.
- Users trust the fit result enough to continue shopping.

## 4. Phase 2: IKAS Pilot MVP

Goals:

- Run Karyamoni on selected products for one or more IKAS pilot merchants.
- Track shopper behavior.
- Learn what merchants need during setup.
- Measure early conversion and return-rate signals.

Deliverables:

- IKAS app installation path.
- Embedded storefront try-on entry.
- Embedded try-on experience.
- Merchant setup dashboard.
- Basic admin console.
- Analytics event pipeline.

Success signals:

- Merchant can activate selected products.
- Shopper flow works on live product pages.
- Try-on events are captured.
- At least one pilot merchant can compare return-rate impact.

## 5. Phase 3: Merchant Readiness

Goals:

- Reduce manual work required to onboard merchants.
- Improve product mapping.
- Improve performance and reliability.
- Strengthen dashboard reporting.

Deliverables:

- Product readiness states.
- Better merchant onboarding.
- Setup checklist.
- Improved analytics dashboard.
- Error diagnostics.

Success signals:

- More merchants can onboard with less team help.
- Product activation errors are visible and fixable.
- Dashboard clearly communicates value.

## 6. Phase 4: Shopify Expansion

Goals:

- Reuse the core try-on platform for Shopify.
- Validate whether the platform adapter approach works.
- Compare IKAS and Shopify merchant needs.

Deliverables:

- Shopify app/adapter.
- Shopify storefront entry.
- Shopify product mapping.
- Platform-specific setup documentation.

Success signals:

- Shopify support does not require rebuilding the core product.
- Product mapping and analytics patterns transfer from IKAS.

## 7. Phase 5: Category and Accuracy Expansion

Goals:

- Add more clothing categories.
- Improve fit guidance accuracy.
- Improve 3D garment realism.
- Support richer shopper profiles.

Possible additions:

- Pants.
- Dresses.
- Jackets and outerwear.
- Full outfit try-on.
- Saved shopper profile.
- More detailed body measurements.
- Better garment simulation.

## 8. Near-Term Backlog

- Build design reference library.
- Confirm IKAS app installation and storefront extension options.
- Create shopper wireframes.
- Create merchant dashboard wireframes.
- Write Turkish-first UI copy.
- Define privacy consent copy.
- Draft analytics event schema.
- Decide first 3D technology direction.
- Define product data requirements for tops.
- Create pilot merchant onboarding checklist.
