# Karyamoni Docs

This folder contains the working product documentation for Karyamoni.

## Documents

- [Product Plan](PRODUCT_PLAN.md): IKAS app-first product vision, MVP scope, market focus, roadmap, and open questions.
- [MVP Spec](MVP_SPEC.md): first release scope, requirements, success criteria, and release criteria.
- [User Flows](USER_FLOWS.md): shopper, merchant, admin, localization, and measurement flows.
- [Technical Architecture](TECHNICAL_ARCHITECTURE.md): early system components, data model draft, integration strategy, and open technical questions.
- [Roadmap](ROADMAP.md): phased path from product definition to IKAS pilot, Shopify expansion, and category growth.
- [Website Plan](WEBSITE_PLAN.md): public website, playground, authentication, localization, dashboard, and design direction.

## Documentation Goals

- Keep the product direction clear.
- Capture decisions before implementation starts.
- Support both product and technical planning.
- Make Turkish and English requirements visible from the beginning.

## Current Planning Priority

The next important work is to validate the IKAS app installation model, collect design references, and turn the embedded shopper and merchant flows into wireframes.
